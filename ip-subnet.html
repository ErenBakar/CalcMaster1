IP Subnet ve CIDR Hesaplama Temelleri

Ağ adresi = IP AND mask. Broadcast = ağ OR NOT mask. Host sayısı = 2^(32-CIDR) − 2. /24 = 254 kullanılabilir host.

CIDR Notasyonu Pratik Kullanım

/30 point-to-point link (2 host), /26 sunucu VLAN (62 host), /24 kullanıcı ağı (254 host).

Private vs Public IP

10.x, 172.16–31.x, 192.168.x internete doğrudan çıkmaz; NAT ile dönüşür.

Ağ Segmentasyonu ve Güvenlik

IoT ve misafir Wi-Fi ayrı subnet'te tutulmalı. Firewall kuralları segment sınırında uygulanır.

IPv6 /64 Standartları

IPv6'da adres kıtlığı yoktur; /64 LAN için standarttır.

Subnet Hesaplayıcı Kullanım Senaryoları

Yeni ofis ağı planlama, VPS firewall kuralı, Docker network ve sınav hazırlığı için anlık ağ/broadcast/host aralığı verir.

Sıkça Sorulan Sorular

/24 subnet kaç host?

254 kullanılabilir host (256 − ağ − broadcast).

192.168.1.100/24 ağ adresi nedir?

192.168.1.0; broadcast 192.168.1.255; host aralığı .1–.254.

/30 ne zaman kullanılır?

Router-to-router point-to-point linklerde; yalnızca 2 host.

Subnet mask ile CIDR aynı mı?

Evet, farklı gösterim: /24 = 255.255.255.0.
